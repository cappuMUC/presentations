Facts

  • Ein System zum Verwalten von Tickets
  • Zugriff über den Webbrowser
  • Stetige Entwicklung seit 2006
  • Open Source Software (GPLv2)
  • Hosting als SaaS, oder eigener Server

Einsatzbereiche

  • Bugtracker für Software
  • Projekt Management
  • Bearbeitung von Support Anfragen
  • Steuern von Prozessen (Workflows)
  • Sammeln von Wissen (Knowledgebase)
  • Lebensläufe von Produkten
  • ...

Features

Tickets

  • Direkter Aufruf über die Ticket Nummer
  • Unterschiedliche Arten von Tickets möglich
  • Eigene Felder & Workflows
  • Benachrichtigungen via Email
  • Tickets erstellen/bearbeiten via Email

Management & Controlling

  • Verwaltung von Benutzern & Gruppen
  • Umfangreiche Filtermöglichkeiten
  • Unterschiedliche Ansichten (Liste, Gantt, Kalender)
  • Sammeln von Tickets in Meileinsteinen (Versionen)
  • Zeiterfassung

Team

  • Wiki zur Dokumentation und Sammlung von Wissen
  • Zentraler Austausch von Dokumenten und Dateien
  • Foren zum Führen von Diskussionen
  • News zur Verbreitung von Informationen im Projekt
  • Browser für Projektarchive (SVN, Git, ...)
  • Volltextsuche über alle Komponenten

Fast jedes Objekt lässt sich referenzieren...

Erweitern

  • Mehr Funktionen über Plugins
  • Anpassungen an UI über Themes
  • Zugriff über API
  • Skripte auf dem Server

&
Cons

Vorteile

  • Sehr flexibel
  • Viele Erweiterungen
  • Eigene Anpassungen möglich
  • Geringe Anschaffungskosten
  • Flache Lernkurve
  • #1234 ist simpel!

Nachteile

  • Schulung von Mitarbeitern
  • Prozesse/Workflows müssen entwickelt werden
  • Das System muss gepflegt werden
